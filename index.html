What is WordPress?

Official description from WordPress.org

WordPress is web software you can use to create a beautiful website or blog. We like to say that WordPress is both free and priceless at the same time.

WordPress is

Popular

WordPress is the #1 content management system in the world.

WordPress is

Free, open source, web-based software

WordPress core software is built by hundreds of community volunteers. Thousands of people around the word contribute to it by either testing the software, submitting bugfixes or building plugins and themes for it.

WordPress is

A website building framework

WordPress allows people to build and design a website with a visual admin and custom code.

WordPress is

A content management system

WordPress allows site owners to manage their site content via an easy-to-use admin.

WordPress.org vs. WordPress.com

  • WordPress.org - Web software used to build a website on a hosting account with a domain name. Sites can be built and modified using themes, plugins or custom code

  • WordPress.com - A free blogging service that allows people to create a blog hosted on WordPress.com's servers. Hosting is free (although there are some for-pay add-ons). Choices of themes are limited, plugins can not be installed and code can not be modified

Installing WordPress

What WordPress needs to run

  • A hosting account running PHP version 5.2.4 or greater and MySQL version 5.0 or greater
  • A domain name
  • Software from WordPress.org or a "one-click install" feature through the hosting account control panel

Installing WordPress

What Is Needed For a Manual Install

  • Access to cpanel (or similar) to create a database and database user
  • A FTP Program. FTP is the method used to upload and download files between a computer and a hosting account
  • A Web browser. Installation is performed step-by-step on a web browser

Security

Being popular means you're a target for hackers

It is important that you take security seriously when running a WordPress site.

Security

During install

  • Make your database table prefix unique. Don't use "wp_" or "wordpress_"
  • Make your database password strong. If your host has a password generator use it.
  • Do not use "admin" as your admin username. "admin" is the default WordPress username and it's used during brute force attacks. Use a unique username and a very strong Password

Security

Before you start: WordPress hardening

  • Take some code-level and sysadmin steps to "harden" WordPress
    • Securing wp-includes
    • Securing wp-config.php
    • Disable File Editing
    • And more
  • Or use a security plugin that performs "1-click hardening"

Security

After your site's live: Monitoring and protection

  • Keep WordPress core, themes and plugins updated!
  • Use a service that scans and protects your site daily from hackers and malware
  • Use a firewall service that shields your site from hackers and malware. Some hosts, like WPEngine.com has this built into their hosting service

Security

If you do get hacked

  • Use a malware removal service.
    • Sucuri.net Reasonably priced WordPress security experts
    • WPEngine.com has this service built into their hosting service

Setting Up Your New Site

The Settings menu

  • General: Title, tagline, site address, date format, etc.
  • Writing: Post-related settings
  • Reading: Set site home page, # of posts, rss settings and search engine visibility
  • Discussion: Manage comment settings
  • Media: Make default image sizes and other media settings
  • Permalinks: Define the structure of your friendly URLs

Exercise time!

Some tasks to get you familiar with the Settings menu and dashboard

  • Change the name of your site
  • Change the "Front page" of your site to the sample page
  • Require users to register before posting a comment
  • Require comments to be approved by an administrator before they are posted
  • Change the dimensions of "Large size" images
  • Remove the "WordPress Blog" and "Other WordPress news" from the dashboard

Changing Site Layout & Design

The Appearance menu

  • Themes: Shows the current theme in use and any other themes installed (more on this on day 2)
  • Widgets: Handy elements you can add to site sidebars or other "widgetized" areas
  • Menus: Awesome menu manager (more on this soon)
  • Header: Manage header text and image
  • Background: Manage background color and image
  • Editor: Dangerous way to edit theme and plugin flies. This is disabled if you "harden" WordPress

Exercise time!

Some tasks to get you familiar with the Appearance menu

  • Remove a few widgets and add some of your own text to the sidebar using a text widget
  • Add a picture to your header
  • Change the color of your header text
  • Change the color of your background

Adding Content

Pages and Posts

  • Pages: Regular single website pages with content
  • Posts: Entries (like blog posts) that have a date, categories and tags assigned to them. They can be listed in a particular order (chronologically, alphabetically, etc.) and fed to other sites or blog readers via an RSS feed

Managing Menus

The Appearance > Menus manager

The Menus manager is awesome, but it requires theme support to work. Twenty Twelve comes with one menu location.

  • Create menus: Name your menu and hit save
  • Add pages: Add existing pages, custom links or category pages
  • Order Pages Just drag and drop your pages in the order you want. Indent pages to make them children of other pages
  • Assign Menu to Theme location When the menu is ready to go live save it, assign it to a theme location and save.

Managing Comments

The Comments menu

Managing Media

The Media menu

View all files that have been uploaded to the site via the content editor. Add, edit and delete files.

Managing Users

The Users menu

  • Manage existing users: Edit all user data or delete users
  • Add new users: Assign new users to site
  • Your Profile: Edit your own profile settings

Managing Users

User roles

Users can be given certain levels of editing privileges.

  • Administrator: access to all the administration features within a single site
  • Editor: ability to publish and manage posts including the posts of other users
  • Author: ability to publish and manage their own posts
  • Contributor: ability to write and manage their own posts but cannot publish them
  • Subscriber: ability to only manage their profile
